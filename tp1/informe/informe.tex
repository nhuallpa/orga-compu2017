\section{{\normalsize Introducción}}

El objetivo del presente trabajo práctico es implementar las funciones encode y decode de datos  base 64 en código assembly. Para ello nos conectamos con el emulador gxemul para realizar la codificación en mips 32 y posteriormente poder realizar las pruebas pertinentes.

\section{{\normalsize Implementación}}

\subsection{{\normalsize Lenguaje}}

Como lenguaje de implementación se eligió ANSI C
ya que el mismo permite una alta portabilidad entre diferentes plataformas. 
El desarrollo del programa se realizó usando un editor de texto 
(gedit,vim, kwrite) y compilando los archivos fuente con 
\htmladdnormallink{GCC}{http://gcc.gnu.org/} que viene en linux.
Para compilar, ejecutar el siguiente comando:

\begin{tabbing}
------- \= ----- \= \kill
\> \textbf{\emph{\$ make}}\\ 
\end{tabbing}

\subsection {{\normalsize Descripción del programa}}

La función \texttt{main} se encuentra en \texttt{tp1.c} 
y se encarga de interpretar las opciones y argumentos. En caso de ser una opción, 
como ayuda o versión, se imprime el mensaje correspondiente y finaliza
la ejecución. Cuando no es una opción de ayuda o versión, se procede 
a procesar los datos de entrada. La salida de estas funciones proveen 
un código de error que sirve como salida del programa. Los mensajes de 
versión y ayuda se imprimen por stdout y el programa 
finaliza devolviendo 0 (cero) al sistema. Los mensajes de error se 
imprimen por la salida de errores (stderr) y el programa finaliza 
devolviendo 1 (uno) al sistema.


\subsubsection {{\normalsize Errores posibles}}

\begin{enumerate}
\item La invocación del programa es incorrecta.
\item Alguno de los archivos es inexistente.
\item Se produjo un error en la lectura del archivo a decodificar.
\item Se produjo un error en la escritura del archivo, donde se encuentra el resultado de la decodificación.
\end{enumerate}

Cuando se produce un error en la codificación o decodificación, se devuelve un código distinto de 0
el cual sirve como indice para ver la descripción del error, la cual se encuentra en el vector de 
errores msgerr. En caso de que se devuelva 0.

\subsection{{\normalsize Desarrollo de actividades}}

\begin{enumerate}
\item Se instaló en un linux un repositorio de fuentes 
\htmladdnormallink{(GIT)}{https://es.wikipedia.org/wiki/Git}
para que al dividir las tareas del TP se pudiese hacer una unión de
los cambios ingresados por cada uno de los integrantes más fácilmente. 
\item Cada persona del grupo se comprometió a que sus cambios en el código
fuente y los cambios obtenidos del repositorio que pudiesen haber subido los
otros integrantes del grupo, sean compilados los sistemas operativos Linux y el NetBSD, asegurando así portabilidad entre plataformas planteada en el enunciado. 
\item Se estableció que todos los integrantes en mayor o menor medida, 
contribuyan en el desarrollo de todas las partes del código para que 
nadie quede en desconocimiento de lo que se hizo en cada sección. 
Una parte se dedico al desarrollo de la función encode, mientras que la otra parte
se focalizó en el desarrollo de la función decode. De esta manera los integrantes del grupo
realizaron la programación en codigo assembly de ambas funciones familiarizandose de esta manera
con el lenguaje. 
\item Se propuso antes de iniciar el desarrollo de cada función, diagramar los stacks frames de las funciones que utilizan el encode y decode, respetando la convención de la abi dada por la catedra a través de la bibliografía brindada por la misma.
\item Para crear el presente informe se debe utilizar el comando make en el directorio informe.
\end{enumerate}

\subsection{{\normalsize Casos de prueba}}
		
	\begin{enumerate}
  		\item Codificamos \textbf{Man} ingresando por stdin y devolviendo por stdout \newline \textbf{TWFuCg==} 
  			\newline Para esta prueba ejecutamos por consola, estando posicionados en la carpeta pruebas, el siguiente comando: 
  			\begin{lstlisting}[language=bash,frame=single]
echo Man | ../tp1 -a encode
			\end{lstlisting}

	  	\item Codificamos \textbf{Man} ingresando por stdin. Luego decodificamos por stdout 
	  		\newline	obteniendo como resultado \textbf{Man}. 
	  		\newline Para esta prueba ejecutamos por consola, estando posicionados en la carpeta pruebas, el siguiente comando: 
	  		\begin{lstlisting}[language=bash,frame=single]
echo -n Man | ../tp1 | ../tp1 -a decode | od -t c
			\end{lstlisting}
	  	\item Verificamos bit a bit, obteniendo por stdout 
	  		\newline \textbf{0000000   x   y   z  \textbackslash n }
	  		\newline \textbf{0000004} 
	  		\newline Para esta prueba ejecutamos por consola, estando posicionados en la carpeta pruebas, el siguiente comando: 
	  		
\begin{lstlisting}[language=bash,frame=single]
echo xyz | ../tp1 | ../tp1 -a decode | od -t c
\end{lstlisting}
	  	\item Codificamos 1024 bytes y chequeamos que no haya más de 76 unidades de longitud. 
	  		\newline \textbf{El resultado es una secuencia de palabras eQp5CnkK que se repiten 9 veces por linea, teniendo un total de 17 lineas.}\newline \textbf{En la última línea, la secuencia finaliza con las palabras eQp5Cg==}\newline Para esta prueba ejecutamos por consola, estando posicionados en la carpeta pruebas, el siguiente comando: 
\begin{lstlisting}[language=bash,frame=single]
yes | head -c 1024 | ../tp1 -a encode | od -t c
\end{lstlisting}
		\item Verificamos que la cantidad de bytes decodificados sea 1024.
			\newline \textbf{1024}
			\newline Para esta prueba ejecutamos por consola, estando posicionados en la carpeta pruebas, el siguiente comando: 
		\begin{lstlisting}[language=bash,frame=single, basicstyle=\tiny]
export ORI=archivo_orig_yes.txt &&
export RES=archivo_result_yes.txt &&
yes | head -c 1024 | tee $ORI | ../tp1 | ../tp1 -a decode | tee $RES | wc -c && 
diff -s $ORI $RES && 
cat $RES | od -t c
		\end{lstlisting}
			\newline Adicionalmente, respaldamos la informacion generada para la entrada y la info resultante en la decodificación para luego verificar que sean iguales. 
			\newline Resultado obtenido:
			\newline \textbf{    1024}			
			\newline \textbf{Files archivo\_orig\_yes.txt and archiv\_result\_yes.txt are identical}			
			\newline \textbf{0000000    y  \textbackslash n   y  \textbackslash n   y  \textbackslash n   y  \textbackslash n   y  \textbackslash n   y  \textbackslash n   y  \textbackslash n   y  \textbackslash n}	
			\newline \textbf{*}	
			\newline \textbf{0002000}	
		\item Codificamos el contenido del archivo de \textbf{entrada.txt}, el cual contiene la palabra \textbf{Man}. Guardamos la salida en el archivo \textbf{salida.txt}, el cual contiene como resultado de la codificación \textbf{TWFuCg==}. Luego decodificamos la salida de este archivo y lo mandamos a otro archivo de entrada (\textbf{entrada2.txt}). Vemos que en este último archivo se encuentra la palabra que se había pasado como entrada en el primer archivo, verificandose de esta manera la codificación y decodificación. 
			\newline Para esta prueba ejecutamos por consola, estando posicionados en la carpeta pruebas, los siguientes comandos: 
\begin{lstlisting}[language=bash,frame=\tiny]
../tp1 -i entrada.txt -o salida.txt -a encode
../tp1 -i salida.txt -o entrada2.txt -a decode
diff -s entrada.txt entrada2.txt 
\end{lstlisting}
		\item Codificamos y decodificamos caracteres con scape.
		\newline Para esta prueba ejecutamos por consola, el siguiente comando.
\begin{lstlisting}[language=bash,frame=single, basicstyle=\tiny ]
export ORI=archivo_orig_esc.txt &&
export RES=archivo_result_esc.txt &&
echo -ne 123tzyx\\t\\n | tee $ORI | ./tp1 | ./tp1 -a decode | tee $RES | od -c &&
diff -s $ORI $RES
\end{lstlisting}
		\item Codificamos y decodificamos un archivo imagen, que se puede encontrar en la carpeta prueba llamado linux-icon.png
		\newline Para esta prueba ejecutamos por consola, el siguiente comando.
\begin{lstlisting}[language=bash,frame=single, basicstyle=\tiny]
./tp1 -a encode -i linux-icon.png | ./tp1 -a decode -o linux-icon.png.b64 &&
diff -s linux-icon.png linux-icon.png.b64
\end{lstlisting}
		\item Corremos el script \textbf{test\_enunciado.sh}. El programa verifica con diff la codificación, mostrando las diferencias en caso de error. Finalmente esperamos superar la centena de corridas para cortar su ejecución haciendo ctrl+c.
				
	\end{enumerate}
